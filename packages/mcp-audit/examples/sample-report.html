Details
Literal credential value(s) found for: NOTES_API_KEY (OpenAI/Anthropic-style key). Values are redacted from this report.
Threat
Client config files are world-readable to local processes, sync to backups and dotfile repos, and get pasted into bug reports. A literal credential in one is a theft waiting for an occasion.
Why this matters
Environment indirection ("${VAR}") or OS keychains keep the secret out of a file that routinely travels. The finding names the variable, never the value.
Remediation
Move the value(s) into your shell environment or OS keychain and reference them from the config (e.g. "${NOTES_API_KEY}").

claude-desktop:example-tools-mcp (/Users/example/Library/Application Support/Claude/claude_desktop_config.json)
Details
example-tools-mcp@0.3.1 has no provenance attestation on the npm registry.
Threat
Without provenance, a tampered or account-takeover release is indistinguishable from a legitimate one at install time; trust rests entirely on whoever controls the npm credentials.
Why this matters
Provenance moves trust from a stealable token to a verifiable build. Once present, 'npm audit signatures' catches substitution mechanically.
Remediation
Prefer servers published with npm provenance (verifiable via 'npm audit signatures'), or pin an audited version and review upgrades manually.

Details
example-tools-mcp@0.3.1 declares one or more of preinstall/install/postinstall, which execute on the host at install time.
Threat
Install hooks run arbitrary code at install time, before the server is ever started: the classic npm supply-chain execution primitive.
Why this matters
MCP servers rarely need native builds. An install hook in a tool-server package deserves a manual look before the package is trusted.
Remediation
Review the package's install scripts before trusting it, or install with scripts disabled ('npm i --ignore-scripts') where the server supports it.
